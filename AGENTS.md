# リポジトリガイドライン

## プロジェクト概要（Overview）

本プロジェクトの目的や背景は以下の通り
- 目的
    - Webサイトの更新チェックを自動化すること
- 背景
    - 人間が手動で巡回することには時間が掛かる。AIを活用することで、従来の変更検知よりも柔軟な分析を実現する。
- 利用範囲
    - 個人利用
    - 生成AI (MCP経由での連携)

## ゴール

- ユーザー(人間)
    - Web UIからサイトのURLとタイトル、AIへの指示をDBに登録できる
    - Web UIから生成AIが調べた結果を確認することができる

- 生成AI
    - MCP 経由でシステムとやりとりできる
    - ユーザーが登録したサイトの情報一覧をDBから取得できる
    - ユーザーの指示でサイトを調べることができる(スクレイピング)
    - 調べる際にはDBから過去の記録も参照しながら変化を調べることができる
    - 調べた結果をDBに登録できる

## 知識＆ライブラリ（Knowledge & Library）

- **MCP (Model Context Protocol)**: AIが外部ツールと通信するためのプロトコル。
- **巡回ワークフロー**: `check_all_sites`を呼び出すと、全サイトの巡回・変更検知・保存が自動で行われる。

## メンテナンス・ポリシー（Maintenance policy）

- ドキュメントは `docs/` 配下に Markdown 形式で残す。
- 変更を加える際は `PLAN.md` を更新し、進捗を管理する。

## ブランチ戦略

- 開発中は `develop` ブランチをベースに作業を行う。
- 機能追加は `feature/xxx` ブランチを作成して行う。
- `master` ブランチは安定版として管理する。

## 生成AIへの指示

- 全て日本語(関西弁)で応答すること。
- 変更を加える際には計画を立て、ユーザーの許可を得てから作業を行うこと。
- ベストな実装方法（クリーンコード、デザインパターン等）を積極的に提案すること。
